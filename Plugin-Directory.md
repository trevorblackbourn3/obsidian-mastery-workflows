## How to Use This Directory

**Organization:** Plugins grouped by category and use case.

**For each plugin:**
- **Name** - Plugin identifier
- **Purpose** - What it does
- **Use case** - When to use it
- **Difficulty** - Beginner / Intermediate / Advanced

**To install:** Settings ‚Üí Community Plugins ‚Üí Browse ‚Üí Search plugin name

   Most popular plugins appear first

### Evaluating Plugins

**Before installing, check:**

1. **Last updated:** Recently maintained?
2. **Download count:** Widely used?
3. **Description:** Does it solve your problem?
4. **Documentation:** Good setup guide?

### Plugin Installation Best Practices

1. **Read documentation first**
2. **Install one at a time** (easier to troubleshoot)
3. **Test for conflicts** (especially with hotkeys)
4. **Disable if unused** (don't just leave it inactive)
5. **Update regularly** (but read changelogs first)

## Plugin Conflicts

**Common plugin conflicts:**

1. **Dataview + Templater:** Both use `<%>` syntax (but compatible)
2. **Multiple hotkey plugins:** Shortcut conflicts
3. **Theme + Style plugins:** CSS conflicts
4. **Multiple calendar plugins:** UI overlap

**Resolution:**
- Disable one
- Adjust settings to avoid conflict
- Use only one plugin for each purpose

---

## Plugin Performance Impact

**Performance-intensive plugins:**

- **Dataview:** Complex queries slow down large vaults
- **Calendar:** Loads all daily notes
- **Graph Analysis:** Processes entire graph
- **Obsidian Git:** Large vaults slow commit

**Optimization:**
- Limit query scope
- Exclude folders from indexing
- Disable plugins you're not actively using
